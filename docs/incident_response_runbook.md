# Incident Response Runbook

## Purpose
Defines steps for responding to cybersecurity incidents.

## Trigger Events
- Suspicious login activity
- Malware detection
- Data exfiltration alerts

## Response Steps
1. Identify and classify the incident
2. Contain affected systems
3. Preserve evidence
4. Eradicate root cause
5. Restore services
6. Document lessons learned

## Escalation
High-severity incidents must be reported to Security Leadership
within one hour of detection.
